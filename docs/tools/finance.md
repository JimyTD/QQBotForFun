# Finance · 经济天气播报

**Status**: v1 (implemented)
**Date**: 2026-06-02

---

## 0. 概述

每工作日自动检测金融市场异动和宏观数据更新，由 LLM 生成一段"大白话"播报推送到群。

**核心理念：**
- 只说「值得说的」—— 统计异动检测，过滤正常波动噪声
- 大白话输出 —— LLM 翻译结构化数据，避免专业术语轰炸
- 被动 + 主动 —— 固定时刻快照 + 群友可手动触发
- 免责 —— 每条播报附"不构成投资建议"

**依赖：** `akshare>=1.18`、`numpy>=1.26`

## 2. 技术设计

### 2.1 异动检测算法

对每个品类的近 30 个交易日绝对涨跌幅计算均值 (μ) 和标准差 (σ)，今日涨跌幅的绝对值超过 **μ + 1.5σ** 时触发异动。

**N=1.5 经回测验证**，约 1.2 条/交易日，平衡了信息量和噪声。

### 2.2 品类覆盖

| 品类 | AKShare 接口 | fetch_kind |
|------|-------------|------------|
| 沪指/深成指/创业板 | `stock_zh_index_daily` (Sina) | `index_sina` |
| 标普500/苹果 | `stock_us_daily` (Sina) | `us_stock_sina` |
| 黄金 | `spot_golden_benchmark_sge` | `gold_sge` |
| 美元/欧元/日元汇率 | `currency_boc_safe` (外管局) | `forex_safe` |

### 2.3 宏观指标

| 指标 | AKShare 函数 | 频率 |
|------|-------------|------|
| CPI月率 | `macro_china_cpi_monthly` | 月度 |
| PPI年率 | `macro_china_ppi_yearly` | 月度 |
| LPR利率 | `macro_china_lpr` | 月度 |

宏观数据检测方式：轮询最新日期与已播报日期对比，有新数据即播报。
已播报状态存 DB（`tool_finance_macro_seen` 表），重启不丢失。

### 2.4 LLM 报告生成

异动检测 + 宏观检测结果 → 结构化文本 → LLM（`finance_report` scene）→ 自然语言播报。

LLM 不可用时自动降级为纯规则拼接的文案。

### 2.5 定时调度

- Cron `30 15 * * 1-5`（工作日 15:30），通过 `core.scheduler.schedule_cron` 注册
- 拉取数据 → 检测 → 生成报告 → 群发到所有已启用群

### 2.6 缓存

内存缓存 5 分钟 TTL。同一次播报任务中多个品类共享外汇 raw data，避免重复请求。

## 3. 数据存储

**表：`tool_finance_groups`**

| 字段 | 类型 | 说明 |
|------|------|------|
| `group_id` | String(32), PK | QQ 群号 |
| `enabled` | Boolean | 是否启用 |
| `enabled_at` | DateTime | 最近一次开启时间 |
| `enabled_by` | String(32) | 开启者 QQ 号 |

**表：`tool_finance_macro_seen`**

| 字段 | 类型 | 说明 |
|------|------|------|
| `indicator_id` | String(64), PK | 宏观指标 ID |
| `last_date` | Text | 最近播报的数据日期 |
| `last_value` | Text | 最近播报的数值 |

## 6. 约定与限制

- 轻量工具，不接入游戏框架
- 群消息通知，不支持私聊
- 零经济交互，不涉及 coin/score
- 静默失败，发送失败只记日志
- AKShare 调用同步（加 `time.sleep` 限速），通过缓存减少重复请求
- 异动检测为纯统计方法（均值 + 1.5 倍标准差），无机器学习组件
