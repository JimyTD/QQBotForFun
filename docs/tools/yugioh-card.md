# 🃏 游戏王查卡（tools/yugioh_card）

- **Status**: Draft v0.1
- **Last Updated**: 2026-05-11
- **Type**: 小工具（tool），**非游戏**（无对局、无输赢、无经济）

---

## 1. 定位与边界

### 1.1 这是什么

为群内游戏王玩家提供快速卡片查询服务：@机器人 + 卡名 / 密码，返回卡片详细信息 + 卡图。交互到此结束。

### 1.2 这不是游戏

**显式不走 `GameBase` 框架**，理由：
- 没有对局概念（一条命令一个结果，不存状态）
- 没有输赢
- 没有经济（查个卡加分没意义）
- 没有多玩家交互（查卡者即结果接收者）

作为 `src/plugins/tools/` 下的**拔插式小工具**独立实现。

### 1.3 代码框架角色

```
src/plugins/tools/
  food/             # 吃什么
  ask_ai/           # 查资料
  reminder/         # 工作提醒
  yugioh_card/      # ⬅️ 游戏王查卡（本文档对象）
    __init__.py     # PluginMetadata 注册
    commands.py     # 命令处理器
    api.py          # YGOProDeck API 封装
    render.py       # 卡片信息格式化
```

## 2. 命令

| 命令 | 别名 | 效果 |
|---|---|---|
| `@机哥 查卡 <卡名>` | `ygo` | 模糊搜索游戏王卡片，返回信息+卡图 |
| `@机哥 查卡 #<密码>` | — | 按密码（passcode）精确查询 |
| `@机哥 随机卡` | `random card` | 随机返回一张游戏王卡片 |

**设计原则：**
- 统一 `@机器人` 触发（`rule=to_me()`）
- `priority=3, block=True`，与其他小工具同级
- 不需要 / 前缀

**不支持（v0.1）：**
- 按属性/种族/星级筛选
- 卡组构筑/饼图查询
- 禁卡表查询
- 猜卡小游戏（未来可做成 games/ 下的独立游戏）

## 3. 数据源

### 3.1 YGOProDeck API（在线）

| 端点 | 用途 |
|---|---|
| `GET https://db.ygoprodeck.com/api/v7/cardinfo.php?fname=<name>&language=zh` | 模糊搜索（中/英/日卡名） |
| `GET https://db.ygoprodeck.com/api/v7/cardinfo.php?id=<passcode>` | 密码精确查询 |
| `GET https://db.ygoprodeck.com/api/v7/randomcard.php` | 随机一张 |

**优势：**
- 免费、无需 API Key
- 数据实时更新（新卡发布即可查）
- 支持中文卡名搜索
- 返回卡图 URL（无需本地存储）
- 零维护

**限制：**
- 需要网络（离线不可用，但 QQ Bot 本身也需要网络）
- 无官方 rate limit 说明，但实测正常使用无问题

### 3.2 不使用本地 cards.cdb

不引入 7MB 的离线数据库，理由：
- 需要定期手动更新
- 卡图需要额外下载/在线拉取
- API 方案零维护且数据更全

## 5. 工作流

### 5.1 Bot 端流程

```
用户发 @机哥 查卡 青眼白龙
  ↓
commands._cmd.handle()
  ↓
解析参数：
  - 以 # 开头 → api.search_by_id(passcode)
  - 否则 → api.search_by_name(name)
  ↓
API 返回结果
  ↓
render.format_card(card_data) → 文字卡片
  ↓
组装消息：
  - render.text_card(...)
  - MessageSegment.image(url=card_image_url)
  ↓
matcher.finish(msg)
```

### 5.2 CLI 端流程

```
用户输入 查卡 青眼白龙
  ↓
cli_adapter.handle()
  ↓
api.search_by_name("青眼白龙")
  ↓
终端打印卡片信息（文字）
  ↓
提示图片 URL（CLI 不显示真图）
```

## 6. 防滥用

- **Per-user cooldown**：同一用户 5 秒内不可重复查卡（避免刷屏）
- **API 超时**：httpx 请求 10 秒超时，超时友好提示
- **结果截断**：模糊搜索只展示第 1 张，不会因为匹配太多而刷屏

---

## 7. 与 CLI-Bot 对齐铁律的关系

遵守 `docs/13-cli-bot-parity.md`：
- ✅ 命令集一致：两端都支持 `查卡 <卡名>` / `查卡 #密码` / `随机卡`
- ✅ 数据一致：两端调同一个 API
- ✅ 文案一致：卡片信息格式相同

**豁免：**
- ⚠️ 图片展示：CLI 只显示 URL、Bot 发真图——终端能力限制
