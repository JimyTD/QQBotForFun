- **Status**: Draft v3（规则讨论中）

## 一、玩法概述

### 模式 A：押注模式（首期实现）

1. 玩家 `@机器人 斗蛐蛐` 开一局
2. 系统在等资源预算下随机生成**红蓝两方阵容**，公布对阵面板
3. **30 秒押注时间**，群友发 `押红` / `押蓝` 下注
4. 入场券 5 金币，等额押注
5. 押注结束，系统模拟战斗，播报战斗过程
6. 结算：赢家平分输家的金币池，所有参与者获得 1 金币参与奖

### 模式 B：单挑模式（首期一起做）

1. 玩家 `@机器人 斗蛐蛐 单挑` 开一局
2. 系统随机抽两个兵种（包含无费用兵种），1v1 单挑
3. 同样可押注

## 二、阵容生成

### 2.1 资源预算（押注模式）

- 预算上限：**1000 资源**（暂定）
- 食物、木材、金币**价值一致**，单位总资源 = food + wood + gold
- 兵种数量 = round(预算 / 单位资源)，四舍五入
- 例：火枪手 75肉+25金 = 100资源 → 1000/100 = 10 个

### 2.2 阵容构成（押注模式）

- 每方 2~4 个兵种
- 兵种池：`is_trainable` 且 `cost > 0` 的兵种
- 排除：船只、建筑马车、英雄、宠物
- 每个兵种分配一部分预算，计算数量
- 保证双方总资源尽量相近（允许±5%误差）

### 2.3 单挑模式

- 两边各 1 个兵种，各 1 个单位
- 兵种池：所有有攻击力的兵种（包含无费用的，如英雄、特殊单位）
- 不考虑资源平衡

### 2.4 面板展示

```

━━━━━━━━━━━━━━━━━━━━━━━━

🔴 红方（总资源 985，人口 12）
  火枪手 ×5    | HP 150 | 远23 近13
  散兵 ×3      | HP 120 | 远15

  长枪兵 ×6    | HP 120 | 近8
  龙骑兵 ×2    | HP 200 | 远22 近10

⏱️ 30秒押注时间！发 "押红" 或 "押蓝"
```

- 人口不参与平衡计算，但展示

## 三、战斗模拟

### 3.1 场地模型

- **一维直线场地**，长度 30 距离单位
- 红方全员起始位置 `pos = 0`，蓝方全员起始位置 `pos = 30`
- 同一方**同兵种小队全部重叠**（无碰撞体积），不同兵种独立位置
- 距离 = |小队A位置 - 小队B位置|

### 3.2 士兵个体

- 每个士兵有**独立 HP**（如 5 个火枪手 = 5 个各 150HP 的个体）
- 每个士兵有**独立攻击 CD 计时**
- 每个士兵有**锁定目标**（一个具体的敌方个体）

### 3.3 时间推进

- 模拟以 **tick** 为单位推进（tick 间隔暂定 0.1 秒）
- 每 tick 依次处理：移动 → 攻击判定 → 死亡清算
- 设置**最大模拟时长**（暂定 60 秒 = 600 tick），超时按剩余总 HP 比判胜

### 3.4 移动规则

- 每 tick 每个小队向最近敌方移动 `速度 × tick_interval` 距离
- **当兵种可以对任何敌方发起攻击时，立即停止移动**
- 具体规则：
  - 有远程/攻城攻击的兵种：与最近敌方距离 ≤ 射程 → **停止移动，开火**
  - 纯近战兵种：与最近敌方距离 ≤ 近战射程（1.5）→ **停止移动，近战**
  - 如果远程兵被敌近战兵贴到 < range_min：不主动后退，切换到近战或缩减伤害模式（见 3.5）

### 3.5 攻击判定

#### 攻击模式选择（优先级，逐个目标判定）

对于同时有远程和近战的兵种，根据与锁定目标的距离：

1. **距离 ≤ 1.5**（近战射程）→ 使用**近战攻击**
2. **1.5 < 距离 ≤ range 且 距离 ≥ range_min** → 使用**远程攻击**
3. **距离 < range_min 且 > 1.5**：
   - 若有近战攻击力 → **继续前进到距离 ≤ 1.5**，然后近战
   - 若无近战攻击力 → **原地缩减伤害远程攻击**（伤害 × 50%）
4. **距离 > range** → 继续移动靠近

> 注：当 range_min ≤ 1.5 时，情况 3 不会发生（从远程模式直接切到近战模式）

#### 纯远程兵种（无近战攻击力）

- 进入射程后停止移动、开火
- 被近身（距离 < range_min）→ 缩减伤害远程攻击（50%）
- 不会主动前进

#### 首次攻击

- **立即开火**（初始 CD = 0），攻击后进入 CD = ROF 秒

#### 缺失 ROF 的处理

- 远程 ROF 缺失 → 默认 3.0 秒
- 近战 ROF 缺失 → 默认 1.5 秒

### 3.6 目标选择

- 士兵**随机选择一个存活敌方个体**作为目标
- **锁定目标直到其死亡**，当次攻击伤害溢出不转火
- 目标死亡后重新随机选择新目标
- 暂不实现智能优先级（纯随机）

### 3.7 伤害计算

#### 公式

```
伤害 = 基础攻击力 × 倍率乘积 × (1 - 目标对应抗性)
伤害最低为 1
```

#### 三种伤害类型与三种抗性的对应关系

AOE3 有三种伤害类型，每种只被对应的抗性减免：

| 伤害类型 | 对应抗性 | 数据字段 |
|---------|---------|---------|
| 远程伤害 (Ranged) | 远程抗性 | `armor_ranged` |
| 近战伤害 (Hand/Melee) | 近战抗性 | `armor_melee` |
| 攻城伤害 (Siege) | 攻城抗性 | `armor_siege`（极少见，数据中暂无） |

**关键含义**：
- 鹰炮的攻城攻击打步兵 → 步兵通常没有攻城抗性 → 0%减免 → 全额伤害
- 这就是为什么炮兵打无甲单位特别疼
- 绝大多数兵种只有近战抗性或远程抗性（或两者之一），极少有攻城抗性
- 数据中同时有双抗的兵种只有 5 个（如阿扎普、利菲迪骑士等）

#### 数据缺失：伤害类型标记

当前数据**没有标记每种攻击的伤害类型**。大多数情况下：
- `attack_ranged` → 远程伤害（吃 `armor_ranged`）
- `attack_melee` → 近战伤害（吃 `armor_melee`）
- `attack_siege` → 攻城伤害（吃 `armor_siege`，几乎无人有）

**特殊情况**（如 Abus Gunner 的远程攻击实际是攻城伤害类型）暂按上述默认规则处理。
后续如需精确可爬取 `damage_type` 字段。

#### 倍率计算

- 目标可能同时属于多个类型（如 [Cavalry, Shock infantry]）
- 攻击者的倍率列表中，所有匹配目标类型的**倍率叠乘**
- 例：攻击者对 Cavalry ×0.5、对 Shock infantry ×2.0 → 最终倍率 = 0.5 × 2.0 = 1.0
- 无匹配倍率时 → 倍率 = 1.0

### 3.8 AOE（范围伤害）

#### 设计原则

- **不考虑兵种的特殊攻击模式**（如炮兵霰弹模式、骑兵践踏模式等），每个兵种只使用其默认/主攻击模式
- 场地保持一维线性设计（评估结论：二维平面对文字播报无额外体验收益，复杂度不值得）

#### 数据来源

- Fandom wiki 的爬虫**没有抓取 AOE 半径**（爬虫中无 area/splash 相关解析）
- **aoe3explorer.com** 有完整的 AOE 数据（如鹰炮 AOE=3）
- 需要写补充爬虫从 aoe3explorer.com 抓取，或手动维护常见 AOE 兵种数据

#### AOE3 原版溅射机制（核实结论，仅供参考）

- 主目标承受完整伤害，不受溅射上限约束
- 溅射以主目标为圆心、DamageArea 为半径
- DamageCap（通常 = 基础攻击 × 2）限制溅射总伤害
- 溅射伤害在范围内单位间随机分配

#### 斗蛐蛐简化 AOE 模型

一维场地 + 同小队重叠，简化规则：

- 有 AOE 属性的兵种攻击时：
  - 主目标受 **100% 完整伤害**
  - **额外溅射目标数上限 = round(aoe_radius)**（四舍五入）
    - 例：鹰炮 aoe_radius=3 → 最多额外溅射 3 个目标 → 共 4 个受伤
  - 溅射目标从**与主目标距离 ≤ aoe_radius 的存活敌方个体**中随机选取
    - 同小队个体距离=0，天然在范围内
    - 不同小队需判断小队间距离是否 ≤ aoe_radius
  - **DamageCap = 基础攻击 × 2**（溅射伤害总量上限）
  - 溅射伤害均匀分配：每个溅射目标 = DamageCap / 实际溅射目标数
  - 溅射伤害同样应用倍率和抗性
  - 主目标的超杀伤害不转移
- **首期可暂不启用 AOE**，先做基础战斗，AOE 数据补充后加入

### 3.9 攻城攻击在斗蛐蛐中的处理

**斗蛐蛐没有建筑，但攻城攻击是许多兵种的主要输出手段。**

处理规则：
- **只有攻城攻击的兵种**（鹰炮、臼炮等 30 个）：
  - `attack_siege` → 视为其远程攻击值
  - `range_siege` → 视为远程射程
  - `rof_siege` → 视为远程 ROF
  - `multipliers.siege` → 视为远程倍率
  - 伤害类型 = **攻城伤害**（吃 armor_siege，绝大多数兵种无此抗性 → 0%减免）
- **同时有远程+攻城的兵种**（如火枪手）：
  - 远程作战时使用 `attack_ranged`（远程伤害类型）
  - 忽略 `attack_siege`（那是打建筑用的）
- **同时有近战+攻城但无远程的兵种**（如某些攻城步兵）：
  - 近战距离使用 `attack_melee`
  - 远距离使用 `attack_siege` 作为远程攻击（如果有 range_siege）
