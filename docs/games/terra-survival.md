## 已拍板的共识（恢复时仍有效）

### 产品形态

- **不是** 一局一关的 `GameSession` 游戏，而是 **群级持久化工具/沙盘**
- 数据挂 `group_id`，**每次操作写库**；无随意过期
- **焦点模式 B**：`@求生` 进入交互 / `@退出` 退出路由（不删档）
- 异步、不强制多人；一人可玩，多人加速

### 交互与技术

- 主路径：**Pillow 状态图 + 消息表情回应当按钮**（NapCat）；文字/数字兜底
- 与现有 `GameRunner` 生命周期解耦；不占用「一群一局」slot
- 可复用能力：群持久化、表情路由、Pillow 面板、贡献/coin 轻联动

### 玩法边界（讨论结论）

**本质定位**：在 QQ 群里是 **文字生存 / 探索推进**，TFC/GT 是 **题材与节点命名**，不是 mod 模拟器。

| ✅ 可以 | ❌ 不做 |
|---|---|
| 探索 / 采集 / 加工 / 建造（少量核心行动） | 合成表摆放、隐藏配方猜谜 |
| 时代与 **机器当科技解锁**（洗矿机 → 增产、解锁副产） | 产线、物流、电压网、逐步 GT 处理链 |
| 配方全透明；里程碑跃迁 | 24h session 超时、通关删档 |
| 部落仓库 + 个人镐 tier（无背包格子） | 金币直接买生存补给 |
| 探索事件、瓶颈文案（缺硫、缺锡） | 还原多方块摆法 |

### 格雷怎么「有点在」

机器 = **科技树节点 + 被动 buff**，例如：

- 解锁 **蒸汽锅炉** → 开放机械类行动
- 解锁 **粉碎机** → 采矿产量 ×2
- 解锁 **洗矿机** → 增产 + 副产物（硫/银）进仓库，供文案/后期节点用

无需合成表，无需玩家操作「粉碎→洗涤→离心」每一步。

### 群峦怎么「有点在」

- 时代名：石器 → 铸造 → 铁器（扩展：彩钢）
- 节点名：坑窑、浇铸、勘矿、高炉、砧锻
- 探索事件：发现矿脉、脉枯、季节文案

## 探索过程中的主要发现

1. **非可视化媒介** 不适合工业模拟；复杂度到 mod 级别时反馈仍是通知级别，易「食之无味」
2. **有趣方向** 应落在：短循环探索、里程碑解锁、群共享进度、一张图说清「下一步」
3. **v5 草案过厚**（13 行动、5 Pillow 页、分叉/工单/营养/季节等）—— 若恢复应 **大砍**，见下节
4. 与现有 bot 游戏（海龟汤、斗蛐蛐、问答）不同：文字生存偏 **慢反馈、无天然结局**，可能永远是「挂件型」玩法

## 若恢复开发：建议路径

**不要从 v5 直接实现。** 建议顺序：

1. **技术 PoC（可选，2～3 天）**  
   群 JSON 持久化 + 一张 Pillow 假面板 + 表情改状态；内容仅「探索 +1 木」  
   → 观察是否有人自发点

2. **若 PoC 有粘性，再写路 A 规格**  
   - **4 行动**：探索 / 采集 / 加工 / 建造  
   - **5～6 里程碑**：石 → 铜 → 青铜 → 铁 →（可选）蒸汽/洗矿  
   - **1 张主面板**；GT 机器 3～4 个 buff 节点即可  
   - TFC/GT 名词从 [调研文档](./terra-survival-tfc-gt-research.md) 取材

3. **仍不确定是否做** → 停在 PoC，不填内容
