## 零、定位

**群级持久化沙盘工具**，不是一局一关的小游戏。

- 数据挂在 `group_id` 上，**随时写库**
- `@机器人 求生` 进入交互焦点，读档并发面板
- `@机器人 退出` 只关路由，**不删档、不结算**
- 无回合限制；可选极长休眠（如 365 天无操作），默认不因时间过期

灵感：**TFC 的生存与手工冶金时代** + **GT 的蒸汽与矿物处理网**。  
具体科技线以调研文档为准，本文档只定**流程与架构**。

---

## 一、设计原则

1. **异步不限时** — 想玩就点，无需等人、无回合计时
2. **不强制多人** — 一人可推完；多人加快，不硬绑协作
3. **操作极简** — 复杂规则进系统；主路径是**点表情选选项**
4. **乐趣在攀科技** — 看清路线、凑瓶颈、解锁下一层；**不猜配方**
5. **NapCat 交互** — 消息 + 表情回应模拟按钮；Pillow 图减少纯文字
6. **持久沙盘** — 矿脉会枯、仓库会堆、时代会升；不是一次性关卡

---

## 二、核心体验

**一张会刷新的行动面板（Pillow 图）+ 消息上的表情按钮。**

面板永远展示：

- 当前时代 / 下一里程碑
- 部落补给与仓库摘要
- 当前可执行行动（材料够/不够一眼可见）

MC mod 在无 GUI 下能保留的趣味：

- 规划下一层要什么
- 凑齐瓶颈资源
- 多步加工有「做出来」的仪式感
- 处理路线取舍（快 vs 副产多）
- 个人工具升级 + 部落设施升级并行

## 三、交互流程

### 3.1 进入 / 退出（焦点模式 B）

| 指令 | 行为 |
|---|---|
| `@机器人 求生` | 加载群存档 → 注册路由 → 发行动面板（图+表情） |
| `@机器人 退出` | 注销路由；**存档已在每次操作时保存** |
| `@机器人 面板` | 在焦点内刷新当前面板（不消耗行动） |

**路由规则**：

- 焦点内：面板消息上的表情回应 → 执行行动
- 焦点内：关键词/数字兜底（`探索`、`1`…）
- 焦点外：表情回应忽略；不消费普通群聊
- 与其他游戏（海龟汤/斗蛐蛐）互不占 `GameRunner` 生命周期；焦点表独立

### 3.2 行动面板 → 二级面板

```
[Bot 发 Pillow 行动面板 + 短消息贴表情]
  🔍探索  ⛏️采集  🔨加工  🏗️建造  📊科技  🔄刷新

玩家点 🔨
  → [Bot 发加工子面板 + 表情]
     每项写清：配方、材料够不够、需什么设施

玩家点对应项
  → 结算 → 刷新主面板
```

- **所有配方透明**；材料不足时点也回明确提示
- 不做配比猜谜；TFC 合金在 UI 上直接写 `铜×9 锡×1`

### 3.3 兜底

表情异常时支持：数字选主面板、`探索`/`科技`/`仓库` 等关键词。

## 四、数据与架构

### 4.1 持久化（与 session 解耦）

```
group_id ──→ TerraWorldRecord
              ├── state_json   # 时代、仓库、建筑、矿脉…
              ├── updated_at
              └── version      #  schema 迁移用
```

- **每次行动后立即写库**（`await persist()`）
- 不依赖 `GameSessionRecord.status=ended`
- 崩溃恢复：读 DB 最新 state 即可

### 4.2 与现有 GameBase 的关系

| 项 | 选择 |
|---|---|
| 注册 | 仍可用 `@register_game` 便于大厅展示 |
| `min_players` | 0 |
| `event_driven` | True（关键词兜底） |
| `serialize_actions` | True（共享仓库互斥） |
| 生命周期 | **无** `runner.end(COMPLETED)` 式终局；焦点层单独管理 |

### 4.3 state 字段（草案）

```python
{
    "era": "casting",              # 当前时代 id
    "milestones": [...],           # 已完成里程碑
    "warehouse": {...},            # 部落仓库
    "buildings": [...],            # 已建成设施
    "veins": [...],                # 已发现矿脉 {type, remaining, rock_layer}
    "supply": {"food": N, "water": N},
    "players": {
        "qq_id": {
            "tool": "bronze_pick",
            "skills": {"mine": 2, "smith": 1},
            "stamina": {"used": 3, "cap": 10, "reset_day": "2026-05-21"},
            "actions_total": 42,
        }
    },
    "steam": {"buffer": 0, "cap": 100},   # GT 层：蒸汽供给
    "panel_message_id": 123456,
    "focus": True,
}
```

## 五、游戏系统（流程层）

### 5.1 部落仓库 + 个人工具（无背包）

| 归属 | 内容 |
|---|---|
| **部落仓库** | 所有原材料、中间品、副产物 |
| **个人** | 当前装备的工具（镐/勘矿镐 tier）；专长等级 |
| **无** | 个人背包格子 |

加工/建造默认从**部落仓库**扣料；产物回仓库。  
工具升级消耗仓库材料，成品绑定个人 —— 「我的青铜镐」决定我能采什么。

### 5.2 生存：部落补给 + 个人体力

**部落补给池**（TFC 生存线简化）：

- 每次行动默认消耗补给（🍖/💧 或合并为「补给值」）
- 设施减负：水井 → 不耗水；农田/灶台 → 不耗食物（对齐 TFC 基建）
- 见底时不死局：探索仍给保底补给，但其他行动效率降

**个人体力**（/async  pacing）：

- 每人每日（或每自然日）N 次**高效行动**（满产出）
- 超出仍可行动，产出 ×0.5 或耗双倍补给
- 不 hard block，避免「手游式体力墙」

### 5.3 勘矿与矿脉

- 探索 / 勘探（需勘矿镐 tier）→ 发现 `{矿种, 岩层, 储量}`
- 定向采集从矿脉扣 `remaining`
- 脉枯 → 需再勘探；制造「沙盘在运转」感

### 5.4 加工与路线（GT 味道）

同种中间品可选路线，例：

- **粉碎**：快速 1×2，无副产
- **洗涤**：慢，+石头粉/硫化物等副产（后期科技才用得上）

面板用图标/颜色区分「快」「丰副产」。

### 5.5 大型建造：群项目（可选）

高炉、砖砌高炉等材料需求大 → 设 **部落捐献进度条**，任何人随时捐仓库材料。  
不强制同时在线；async 自然协作。

## 六、科技内容（指向调研，不在此定数值）

**MVP 里程碑**见 [调研 §4.2](./terra-survival-tfc-gt-research.md#42-里程碑列表建议-mvp-范围)：

> 石制工具 → 坑窑 → 铜/青铜浇铸 → 勘矿 → 高炉/钢 → 蒸汽锅炉 → 粉碎机 → 洗矿 → 砖高炉 …

已废弃的错误设定（勿再使用）：

- ~~电气时代 / 信号塔通关~~
- ~~篝火→发电机直线~~
- ~~猜铜锡配比~~
- ~~24 小时 session 超时~~
