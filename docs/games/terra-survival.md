# 群峦求生 —— 设计文档

- **Status**: Draft v3
- **Last Updated**: 2026-05-21
- **Game ID**: `terra_survival`
- **灵感来源**: TerraFirmaCraft（生存与加工链） + GregTech（科技分层与瓶颈规划）

---

## 零、设计原则

1. **异步不限时**：没有回合计时，想玩就点
2. **不强制多人**：一人可通关，多人只是更快
3. **操作极简**：复杂规则藏在系统里，玩家只点选项
4. **乐趣在攀科技**：不是猜配方，是看清路线、凑资源、解锁下一层
5. **NapCat 原生交互**：用「消息 + 表情回应」模拟按钮，文字/数字仅作兜底

---

## 一、核心体验（一句话）

**一张会刷新的「行动面板」，上面贴好几个表情；你点哪个，部落就执行哪件事。面板永远告诉你：现在能做什么、离下一层科技还差什么。**

MC / 群峦 / 格雷的真正乐趣是：

- 我知道下一步要造什么
- 我知道还缺什么材料
- 我把它做出来，世界突然多了一整片新玩法

不是「铜锡比例到底是 9:1 还是 8:2」这种无 UI 环境下的劝退题。

---

## 二、交互：表情回应当按钮

### 2.1 为什么这样做

QQ 官方 Bot 有 InlineKeyboard；我们用 NapCat，**不能发真按钮**，但可以：

1. 机器人发一条「选项消息」
2. 机器人给这条消息贴上预设表情（`set_msg_emoji_like`）
3. 玩家点表情 → 触发 `group_msg_emoji_like` 事件（仅机器人自己消息上的回应会推送）
4. 根据 `user_id` + `emoji_id` 执行对应行动

体验上接近「点按钮」，且**不需要打字**。

### 2.2 行动面板（主界面）

开局和每次行动结束后，机器人发/刷新**行动面板**：

```
━━━ 🪨 石器时代 · 行动面板 ━━━
🍖 8  💧 6  |  🪨12 🪵9 🟠4 ⚪2

🎯 下一里程碑：青铜工具（还差 🟤青铜锭×1）
   青铜锭：🟠×9 + ⚪×1，需 🔥篝火

👇 点表情执行（也可发数字兜底）
```

预设表情（示例映射，实现时查 QQ emoji_id）：

| 表情 | 含义 |
|---|---|
| 🔍 | 探索 |
| ⛏️ | 采集 |
| 🔨 | 加工 |
| 🏗️ | 建造 |
| 📊 | 科技树 |
| 🔄 | 刷新面板 |

**规则**：玩家在自己想操作的时候，去点**最新一条**行动面板上的表情即可。不限时，不等人。

### 2.3 二级菜单（仍然用表情）

点 🔨 加工 后，机器人发**子面板**（同样贴表情）：

```
🔨 可加工（材料够的会高亮）：
🟤 青铜锭  🟠9+⚪1  [🔥篝火✓]
⚒️ 石制工具  🪨3+🪵2

👇 点对应表情确认
```

- **所有配方写清楚**，材料够/不够、设施有没有，一眼可见
- 没有猜配比、没有隐藏合成表
- 材料不够时，该选项仍可点，但回复「材料不足，还差 🟠×3」

### 2.4 兜底

表情异常或用户习惯打字时，仍支持：

- 发 `1`~`6` 对应主面板
- 发 `探索` / `状态` / `科技` 等关键词

但**主路径是点表情**，文字是备用。

## 三、玩法循环（攀科技，不是猜谜）

```
看面板 → 知道缺什么 → 探索/采集补资源 → 加工中间品 → 建造设施 → 解锁新时代 → 面板出现新选项
```

### 3.1 科技树是 UI，不是隐藏数据

`📊 科技树` 永远可读，展示：

```
🪨 石器 ──✅──> 🟤 青铜 ──🔒──> ⚔️ 铁器 ──🔒──> 💨 蒸汽 ──🔒──> ⚡ 电气 ──🔒──> 📡 通关
                      ↑ 你在这里
下一解锁：青铜工具
  需要：🟤青铜锭×1 + 🔥篝火
  解锁：青铜时代、铁矿采集、熔炉建造
```

玩家始终知道：

- 我在哪一层
- 下一层门槛是什么
- 解锁后会多出什么

这就是格雷「电压等级」和群峦「时代跃迁」在文字环境里的等价物。

### 3.2 加工链（透明、有层次）

| 阶段 | 中间品 | 配方（全部可见） | 需要设施 |
|---|---|---|---|
| 石器 | 石制工具 | 🪨×3 🪵×2 | 无 |
| 青铜 | 青铜锭 | 🟠×9 ⚪×1 | 🔥篝火 |
| 青铜 | 青铜工具 | 🟤×2 🪵×2 | 🔥篝火 |
| 铁器 | 铁锭 | 🔴×3 ⚫×1 | 🔥熔炉 |
| 铁器 | 铁制工具 | ⚔️×2 🪵×2 | ⚒️铁匠铺 |
| 蒸汽 | 蒸汽零件 | 🔩×2 🟤×1 | 🔬工坊 |
| 电气 | 电子元件 | 🟤×1 🟡×1 | 🏭工厂 |

**乐趣来源**：

- 先造篝火，才能冶炼 → 设施是科技节点
- 先有青铜工具，才能高效采铁 → 工具是时代门票
- 中间品不能跳步 → 加工链有规划感

### 3.3 探索的角色

探索 ≠ 猜配方。探索负责：

- 随机补资源（效率略低于定向采集）
- **首次发现矿脉**（解锁「采集铜/锡/铁」选项）
- 小概率好事件（+资源、+士气）

探索是「开地图」，不是「开合成表」。

## 四、时代与胜利

| 时代 | 进入条件 | 新解锁 |
|---|---|---|
| 🪨 石器 | 开局 | 基础采集、篝火、石墙 |
| 🟤 青铜 | 做出 **青铜工具** | 熔炉、铁矿/煤炭采集 |
| ⚔️ 铁器 | 做出 **铁制工具** + **铁匠铺** | 工坊、钢、金矿 |
| 💨 蒸汽 | 建成 **蒸汽机** | 工厂、石油 |
| ⚡ 电气 | 建成 **发电机** | 信号塔 |
| 🏆 通关 | 建成 **📡 信号塔** | 结算 |

每次升时代：**大卡片播报 + 刷新行动面板**，让玩家感到「世界变大了」。

## 五、建筑（科技节点，不是装饰）

每个建筑只建一次，作用是**解锁加工/采集/减负**：

| 建筑 | 材料 | 作用 |
|---|---|---|
| 🔥 篝火 | 🪵×3 | 解锁青铜加工 |
| 💧 水井 | 🪨×6 🪵×3 | 行动不再耗水 |
| 🌾 农田 | 🪵×5 🧱×3 | 行动不再耗食物 |
| 🔥 熔炉 | 🪨×8 🧱×5 🟤×2 | 解锁铁锭 |
| ⚒️ 铁匠铺 | ⚔️×4 🪨×8 | 解锁钢、铁制工具 |
| 🔬 工坊 | ⚔️×6 🪵×10 | 解锁蒸汽零件 |
| 💨 蒸汽机 | ⚙️×2 🔩×3 | 进入蒸汽时代 |
| 🏭 工厂 | 🔩×8 ⚙️×3 | 解锁电子元件 |
| ⚡ 发电机 | 💡×2 🔩×4 | 进入电气时代 |
| 📡 信号塔 | 💡×4 🔩×6 🟤×4 | 通关 |

**生存线（TFC）与科技线（GT）的取舍**：

- 前期会纠结：先造篝火走科技，还是先造水井/农田省消耗？
- 这是有意义的规划，不是猜谜

---

## 六、生存（轻量，不劝退）

- 每次行动默认消耗 1🍖 1💧
- 建了水井/农田后对应消耗归零
- 资源见底时：**仍可探索**（探索必给少量🍖💧），永不死局
- 不设「第 N 天必须怎样」——异步友好

---

## 七、多人

- 共享一个部落、一个资源池、一棵科技树
- 谁都可以点面板上的表情
- `serialize_actions = True`，避免两人同时扣材料
- 同一玩家 10 秒冷却（防连点刷屏）
- 不强制分工；面板上的「下一里程碑」自然引导协作

---

## 八、与 v2 的主要差异

| v2（已废弃） | v3（当前） |
|---|---|
| 猜合金配比 | 配方全透明，材料/设施写清楚 |
| 发词 + 回数字 | **表情面板为主**，数字兜底 |
| 乐趣在「发现配方」 | 乐趣在「攀科技树 + 凑瓶颈资源」 |
| 多轮对话状态机 | 面板消息 + 表情事件驱动 |

## 九、实现要点

### 9.1 新增能力（项目里尚未使用）

- `set_msg_emoji_like`：给面板消息贴选项表情
- 监听 `notice.group_msg_emoji_like`：解析玩家选择
- 维护 `panel_message_id`：指向当前有效的主面板

### 9.2 GameBase

```python
id = "terra_survival"
min_players = 0
event_driven = True
serialize_actions = True
```

### 9.3 状态字段（精简）

```python
{
    "era": 1,
    "resources": {...},
    "buildings": [...],
    "discovered_ores": [...],      # 探索解锁的矿脉
    "panel_message_id": 123456,    # 当前主面板
    "sub_panel": {...},            # 某玩家打开的二级面板（可选）
    "player_actions": {...},
    "total_actions": 0,
}
```

### 9.4 文件结构

```
terra_survival/
├── game.py           # 主逻辑 + 表情事件入口
├── panel.py          # 行动面板生成 + 表情映射
├── tech_tree.py      # 时代/解锁/里程碑
├── recipes.py        # 加工配方（只读配置）
├── buildings.py
├── world.py          # 资源增减、行动结算
└── emoji_handler.py  # group_msg_emoji_like 监听
```
