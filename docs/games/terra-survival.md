# 群峦求生 —— 设计文档

- **Status**: Draft v2
- **Last Updated**: 2026-05-21
- **Game ID**: `terra_survival`
- **灵感来源**: TerraFirmaCraft（群峦传说） + GregTech（格雷科技）

---

## 零、设计原则

1. **异步不限时**：没有回合计时器，玩家想什么时候操作就什么时候操作
2. **不强制多人**：一个人也能玩完全程，多人只是更快
3. **操作会话简单**：核心操作都是一个词，有选项时发数字即可

---

## 一、玩法概述

QQ 群共享一个"部落世界"。玩家随时发送简单指令（探索、采集、合成、建造）来推进文明发展。
从石器时代出发，目标是一路发展到电气时代并建成信号塔。

### 核心循环

```
玩家发指令 → 消耗食物/水 → 执行行动 → 获得资源/成果 → 推进文明
```

### 一局时长

- 不设时限，按玩家活跃度自然推进
- 参考：单人约 40-60 次操作通关，多人分摊更快
- 唯一硬限制：整局超时 **24 小时**（防僵尸局）

## 二、交互设计

### 2.1 开局

```
@机器人 求生          → 创建游戏，立即可玩（无需等人）
```

### 2.2 游戏内指令

**所有指令不需要 @机器人**，直接发送即可（`event_driven=True`）。

| 指令 | 说明 | 消耗 |
|---|---|---|
| `探索` | 随机获得资源或发现 | 1🍖 1💧 |
| `采集` | 定向采集（系统给选项，回数字） | 1🍖 1💧 |
| `合成` | 合成物品（系统给选项，回数字） | 1🍖 |
| `建造` | 建造设施（系统给选项，回数字） | 1🍖 1💧 |
| `状态` | 查看部落全貌 | 免费 |
| `背包` | 查看资源清单 | 免费 |
| `配方` | 查看已知配方 | 免费 |

### 2.3 选择交互（两轮对话）

当指令需要选择时，系统弹出编号列表，玩家**直接回数字**：

```
玩家: 采集

Bot:  ⛏ 可采集的资源：
      1. 🪨 石头
      2. 🪵 木材
      3. 🟠 铜矿
      请发数字选择：

玩家: 3

Bot:  ⛏ 采集铜矿 → 🟠铜矿 +3
      (🍖-1 💧-1)
```

### 2.4 合金配比交互（核心乐趣）

首次合成合金时，需要猜配比（TFC 精髓）。同样是**选数字**：

```
玩家: 合成

Bot:  🔨 可合成：
      1. 石制工具 (🪨×3 🪵×2)
      2. 🟤青铜锭 (🟠铜矿 + ⚪锡矿)
      请发数字选择：

玩家: 2

Bot:  🟤 青铜是铜和锡的合金。需要大量铜、少量锡。
      请选择 铜:锡 配比：
      1.  6:4
      2.  7:3
      3.  8:2
      4.  9:1
      5. 10:0

玩家: 4

Bot:  ✅ 配比 9:1 → 青铜锭×1！完美配比，已记入配方本。
      (🟠铜矿-9 ⚪锡矿-1 🍖-1)
```

配方一旦发现，后续合成**自动使用已知配比**，一步完成。

## 三、资源与生存

### 3.1 生存机制

**每次行动消耗**食物和水（非时间消耗），自然形成生存压力：

- 食物或水归零 → **只能做** `探索`（探索必定产出少量食物/水，保底不死局）
- 食物和水充足 → 全部行动解锁

没有"饿死"机制——设计哲学是**永不死局**，只是效率降低。

### 3.2 资源表

**基础资源**（石器时代可得）：

| 资源 | 获取 |
|---|---|
| 🍖 食物 | 探索(随机) / 采集 |
| 💧 水 | 探索(随机) / 采集 |
| 🪨 石头 | 探索 / 采集 |
| 🪵 木材 | 探索 / 采集 |
| 🧱 粘土 | 探索 / 采集 |
| 🟠 铜矿 | 探索(低概率) / 采集(需发现矿脉) |
| ⚪ 锡矿 | 探索(低概率) / 采集(需发现矿脉) |

**进阶资源**（需要对应时代/设施解锁采集）：

| 资源 | 解锁条件 |
|---|---|
| 🔴 铁矿 | 青铜时代 + 青铜镐 |
| ⚫ 煤炭 | 青铜时代 |
| 🟡 金矿 | 铁器时代 + 铁镐 |
| 🛢️ 石油 | 蒸汽时代 + 钻机 |

### 3.3 加工材料

| 材料 | 配方 | 需要设施 |
|---|---|---|
| 🟤 青铜锭 | 🟠铜矿 + ⚪锡矿 (配比发现制) | 篝火 |
| ⚔️ 铁锭 | 🔴铁矿×3 + ⚫煤炭×1 | 熔炉 |
| 🔩 钢锭 | ⚔️铁锭×2 + ⚫煤炭×1 | 铁匠铺 |
| ⚙️ 蒸汽零件 | 🔩钢锭×2 + 🟤青铜锭×1 | 工坊 |

### 4.1 五个时代

## 六、探索系统

`探索` 是最核心的行动——随机性带来乐趣和惊喜。

### 6.1 探索产出

每次探索随机获得 **1-3 种资源**，资源种类和数量受时代影响。

基础池（石器时代）：
- 常见：🍖食物(1-3)、💧水(1-3)、🪨石头(1-3)、🪵木材(1-3)、🧱粘土(1-2)
- 稀有(20%)：🟠铜矿(1-2)、⚪锡矿(1-2)

每升一个时代，基础产出 +1，并加入新资源到池中。

### 6.2 探索事件（10% 概率）

好事件：
- 🗺️ "发现了一个矿脉！" → 随机矿物 +5
- 📜 "找到了一块刻满符号的石板…" → 揭示一个未知配方
- 🎁 "发现了前人留下的物资！" → 获得当前时代+1 的材料

坏事件（建了石墙后免疫）：
- 🐺 "遭遇野兽！" → 食物 -2
- 🌩️ "突发暴风雨！" → 木材 -2

## 七、合金配比系统（核心乐趣）

### 7.1 设计理念

**第一次**合成某种合金时需要猜配比（致敬 TFC），但交互方式是**选数字**而非输入复杂命令。
**猜对后**记入配方本，后续一键合成。

### 7.2 合金配比表

| 合金 | 正确配比 | 选项干扰项 |
|---|---|---|
| 青铜 | 铜:锡 = 9:1 | 6:4 / 7:3 / 8:2 / **9:1** / 10:0 |
| 黑铜 | 铜:金 = 6:4 | 5:5 / **6:4** / 7:3 / 8:2 / 9:1 |
| 钢 | 铁:煤 = 7:3 | 5:5 / 6:4 / **7:3** / 8:2 / 9:1 |
| 黑钢 | 钢:黑铜 = 6:4 | 5:5 / **6:4** / 7:3 / 8:2 / 9:1 |

### 7.3 失败惩罚

- 选错配比：材料消耗 50%（不全废，比 TFC 宽松）
- 系统给出提示："比例不太对，需要更多的铜"（方向提示，下次更容易猜对）

## 八、多人协作

### 8.1 协作方式

- **共享世界**：所有玩家操作同一个部落，资源共享
- **无角色分工**：不强制职业，谁想干什么就干什么
- **自然协作**：一个人采矿，另一个人立刻可以用矿合成——先到先得

### 8.2 人数缩放

- **1 人**：完全可玩，自己一步步推进
- **多人**：推进更快，但食物/水消耗也更快（每人每次行动都消耗）
- 建了水井和农田后消耗归零，多人纯加速

### 8.3 行动冷却

- 同一玩家连续操作间隔 **10 秒**（防刷屏，也给其他群友操作空间）
- 不同玩家之间无限制

## 十一、实现要点

### 11.1 文件结构

```
src/plugins/games/terra_survival/
├── __init__.py          # 插件注册
├── game.py              # GameBase 子类，状态机
├── commands.py          # NoneBot 指令路由
├── world.py             # 世界状态（资源、建筑、时代）
├── recipes.py           # 配方定义 + 配比校验
├── events.py            # 随机事件
├── buildings.py         # 建筑定义
└── broadcaster.py       # 播报格式化
```

### 11.2 GameBase 配置

```python
id = "terra_survival"
name = "群峦求生"
min_players = 0          # 不限人数，开局即玩
max_players = 50
event_driven = True      # 群消息转 on_player_action
serialize_actions = True  # 串行处理（共享资源池需要互斥）
```

### 11.3 状态机

```
PLAYING          主状态，处理所有行动指令
  ├─ SELECTING   等待玩家选择（采集/合成/建造的编号）
  └─ RATIO       等待合金配比选择
ENDED            通关或超时
```

`SELECTING` / `RATIO` 是**针对单个玩家**的子状态——该玩家在选择时，其他玩家仍可自由行动。

### 11.4 ctx.state 关键字段

```python
{
    "era": 1,                              # 当前时代 1-5
    "resources": {"food": 5, "water": 5, "stone": 3, ...},
    "buildings": ["campfire", "stone_wall"],
    "recipes_known": ["bronze"],           # 已知合金配方
    "discovered_ores": ["copper", "tin"],   # 已发现矿脉（可采集）
    "total_actions": 12,                   # 总行动数
    "player_actions": {"123456": 5, ...},  # 每人行动数
    "pending_select": {                    # 等待选择（per player）
        "123456": {"type": "gather", "options": [...], "expires": ...}
    },
}
```

### 11.5 关键设计决策

- **`serialize_actions = True`**：因为共享资源池，防止两人同时消耗导致负数
- **选择超时**：玩家发了 `合成` 但不选编号，30 秒后自动取消该选择
- **永不死局**：食物/水归零只限制行动种类，探索保底产出食物/水
- **配方记忆在世界级别**：一个人发现配方，所有人都能用
