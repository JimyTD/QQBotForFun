# 趣味问答 · 题库生成指南（供多 Agent 协作使用）

> **目的**：本文档为任何参与生成题库的 Agent 提供完整规范。严格按照本文档的格式、校验规则和内容要求生成题目，确保最终数据可直接合并使用。

---

## 1. 数据格式（必须严格遵循）

每条题目是一个 JSON 对象，最终文件是 JSON 数组：

```json
{
  "answer": "答案（最标准、最广为人知的主名称）",
  "aliases": ["英文名", "常见别名/简称"],
  "clue_sets": [
    ["线索1(最难)", "线索2", "线索3", "线索4", "线索5(最简单送分)"],
    ["第2套线索1(最难)", "线索2", "线索3", "线索4", "线索5(最简单送分)"]
  ],
  "explanation": "一句话科普（20-60字），答对后展示",
  "difficulty": "easy 或 medium",
  "source": "claude_gen_2026-05-06"
}
```

### 字段要求

| 字段 | 要求 |
|---|---|
| `answer` | 3-15字，玩家最可能脱口而出的名称 |
| `aliases` | 2-5个别名/英文名/简称，**不得与 answer 相同** |
| `clue_sets` | **恰好 2 套**，每套**恰好 5 条**，从难到易排列 |
| `explanation` | 20-60字，有信息量的一句话科普 |
| `difficulty` | `"easy"` 或 `"medium"`，配比约 70% easy / 30% medium |
| `source` | `"claude_gen_2026-05-06"` |

---

## 2. 铁律（违反任何一条 = 不合格）

### 2.1 Self-Leak 禁令（最重要！）

> **answer 和所有 aliases 中的任何一个词（≥2字），都不能出现在任何一条线索中。**

示例：
- ❌ 答案"巴西"，线索"巴西利亚是首都" → "巴西"出现在线索中
- ❌ 答案"蒙古"，别名含"蒙古"，线索"蒙古包是传统住所" → 泄漏
- ❌ 答案"东京"，别名含"江户"，线索"从江户时代发展而来" → 泄漏
- ✅ 答案"蒙古"，线索"圆形毡帐（Ger）至今仍是牧民传统住所" → OK

**常见陷阱**：
- 答案名出现在复合词中（"洛阳水席"泄漏了"洛阳"）
- 别名作为形容词使用（"日式拉面"中的"拉面"如果是别名就泄漏）
- 别名出现在"XX博物馆/XX大学/XX节"等专有名词中

### 2.2 事实准确性

- **所有线索必须基于真实事实**，不得编造
- 宁可写得平淡也不要虚构不存在的"世界之最"、"著名传说"等
- 如果不确定某事实是否准确，换一个确定的角度

### 2.3 线索语言

- **所有线索（clue_sets 中的每一条）必须用中文书写**，不得夹杂英文单词、外文字母、外文短语。
- 可以出现的非中文：阿拉伯数字（年份、数量）、标点符号、极少量的通用符号（如 °C、%）。
- `answer` 也应使用中文（最标准、最广为人知的中文名称）。
- `aliases` **允许包含英文名/外文名/拉丁学名**——这是别名字段的合法用法，仅用于答案匹配与展示；**但这些别名同样受 §2.1 Self-Leak 禁令约束**，即线索里不能出现这些英文/外文片段（≥2 个连续字母也算泄漏）。

### 2.4 不重复

- 同一类型内答案不能重复
- 检查本文档末尾的**已有答案清单**，不要生成已经存在的

---

## 3. 线索质量要求

### 从难到易的梯度

| 位置 | 难度 | 角度举例 |
|---|---|---|
| 线索1 | 最难 | 冷知识、历史典故、侧面描述 |
| 线索2 | 较难 | 文化特征、代表性事物 |
| 线索3 | 中等 | 外观特征、标志性元素 |
| 线索4 | 较易 | 地理位置、所属分类 |
| 线索5 | 送分 | 直接描述身份/位置/类别 |

### 两套线索的差异化

- 第1套和第2套必须**从不同角度**描述同一个答案
- 如果第1套侧重历史/文化，第2套侧重现代/日常
- 如果第1套侧重外形/味道，第2套侧重产地/做法
- 不能只是换了措辞但说的是同一件事

### 线索长度

每条线索 15-50 字，不要太短（无信息量）也不要太长（读起来累）

### 4.4 idiom（猜成语）— 需要 80 道

**目标总量**：80 道（全部是中文有典故的成语）

**范围要求**：
- ✅ 必须有典故/有故事的成语
- ✅ 示例：完璧归赵、刻舟求剑、画蛇添足、守株待兔、卧薪尝胆、破釜沉舟、纸上谈兵
- ❌ 禁止：纯描述性成语（一丝不苟、根深蒂固、井井有条、栩栩如生）

**线索角度**：
1. 故事的结局或主人公心境（最难）
2. 主人公身份（不说名字）、时代背景
3. 故事情节的关键动作或道具
4. 成语的字面意思
5. 寓意 + 出处书籍（《史记》《战国策》等）

**特别注意**：
- ⚠️ 线索中不能出现成语中**任意 2 个及以上连续的字**（与 §2.1 总则对齐）。单个字允许出现——例如成语"背水一战"，线索里可以出现"战"或"水"的单字，但不能出现"背水""一战""水一"这样的连续片段。
- ⚠️ 不能出现"四字成语""这个成语叫""这则典故"等 meta 提示
- aliases 对成语建议给**空数组** `[]`——因为任何别名（如"xxx的故事"中的"故事"二字）会立刻变成本条及其它条的字级约束，成本高；成语本身就是最标准称谓，不需要别名
- 放宽后的校验脚本参考：`any(ans[i:i+2] in clue for i in range(len(ans)-1))` 命中才算泄漏

```
  animal.json    ← ✅ 已完成 80 道

```

### 5.2 生成方式

1. 将你生成的题目写为 **合法 JSON 数组** 文件（如 `_food_batch_new.json`）
2. 放入 `seeds/trivia_bank/` 目录
3. 运行合并校验脚本（见下方）

### 5.3 校验脚本

生成后必须通过以下校验才算合格：

```python
import json
from pathlib import Path
from collections import Counter

def validate_bank(file_path):
    data = json.loads(Path(file_path).read_text(encoding="utf-8"))
    print(f"Entries: {len(data)}")
    
    issues = []
    for i, entry in enumerate(data, 1):
        # 1. 结构检查
        assert "answer" in entry and "aliases" in entry
        assert "clue_sets" in entry and len(entry["clue_sets"]) == 2
        for cs in entry["clue_sets"]:
            assert len(cs) == 5, f"#{i} {entry['answer']}: clue_set has {len(cs)} clues, need 5"
        
        # 2. Self-leak 检查（最重要！）
        all_names = [entry["answer"]] + entry.get("aliases", [])
        for si, cs in enumerate(entry["clue_sets"]):
            for ci, clue in enumerate(cs):
                for name in all_names:
                    if name and len(name) >= 2 and name in clue:
                        issues.append(f'{i}. {entry["answer"]} set{si+1}/clue{ci+1}: "{name}" leaked')
    
    # 3. 去重检查
    answers = [e["answer"] for e in data]
    dups = [a for a, c in Counter(answers).items() if c > 1]
    
    print(f"Self-leaks: {len(issues)}")
    for x in issues:
        print(f"  {x}")
    print(f"Duplicates: {dups if dups else 'none'}")
    
    return len(issues) == 0 and not dups

# 使用方式
validate_bank("seeds/trivia_bank/food.json")
```

### 5.4 合并到已有文件

```python
import json
from pathlib import Path

BANK = Path("seeds/trivia_bank/food.json")  # 改成目标文件
BATCH = Path("seeds/trivia_bank/_food_batch_new.json")  # 你的新生成文件

existing = json.loads(BANK.read_text(encoding="utf-8")) if BANK.exists() else []
batch = json.loads(BATCH.read_text(encoding="utf-8"))

# 去重
existing_answers = {e["answer"] for e in existing}
batch_clean = [e for e in batch if e["answer"] not in existing_answers]

data = existing + batch_clean
BANK.write_text(json.dumps(data, ensure_ascii=False, indent=2), encoding="utf-8")
print(f"Merged: {len(existing)} + {len(batch_clean)} = {len(data)}")
```

## 6. 质量检查清单（每批生成后自检）

- [ ] JSON 格式合法（能被 `json.loads` 解析）
- [ ] 每条有 answer、aliases、clue_sets(2套x5条)、explanation、difficulty、source
- [ ] **Self-leak 检查通过**（answer 和 aliases 不出现在任何线索中）
- [ ] 无重复答案
- [ ] 线索从难到易有明显梯度
- [ ] 两套线索角度不同
- [ ] 事实准确（不编造）
- [ ] explanation 有信息量（20-60字）
- [ ] difficulty 分布合理（约 70% easy / 30% medium）
