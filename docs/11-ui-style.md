# 11 · UI / 文本排版规范

- **Status**: Draft v1
- **Last Updated**: 2026-04-28
- **Owner**: @owner

> 本项目采用**全文本 UI**。一切展示能力由 `core.render` 模块封装为文本排版原语。
> 本文规定 emoji、分隔符、卡片、列表、结果页的统一风格。**所有插件必须通过 `core.render` 输出重要消息**，以保证风格一致。

---

## 1. 设计原则

1. **信息优先**：排版服务于内容，不为装饰而装饰
2. **移动端友好**：单行 ≤ 26 个中文字符（约 52 半角）
3. **可复制性**：不使用过多特殊字符，便于玩家复制回顾
4. **节流**：一次交互最多输出 1 张"卡片" + 必要附加，避免刷屏
5. **最多两级视觉层次**：标题 / 正文，不搞三级以上嵌套

## 2. 字符预算

| 类型 | 软上限 | 硬上限 | 超出处理 |
|---|---|---|---|
| 单条消息 | 800 字 | 2000 字 | 拆分多条 或 合并转发 |
| 单行 | 26 中文字 | 30 中文字 | 手动换行 |
| 卡片正文段落 | 3-5 行 | 8 行 | 截断 + "…" |
| 列表项数 | 10 | 20 | 分页 `(1/3)` |

## 3. Emoji 使用规范

**原则：有意义、一致、克制**。每类场景使用**固定** emoji，不随意换。

### 3.1 语义 emoji（含义固定，不得滥用）

| Emoji | 语义 | 用法 |
|---|---|---|
| ✅ | 成功 / 是 | 判定通过、命令成功 |
| ❌ | 失败 / 否 | 判定失败、权限不足、输入非法 |
| ⚠️ | 警告 | 边界提示、余额不足等 |
| ❓ | 问题 / 提问 | 玩家提问的前缀 |
| 💡 | 提示 / 线索 | 关键线索、操作提示 |
| ⏱ | 时间 / 计时 | 超时、计时提示 |
| 🎮 | 游戏 | 游戏大厅、游戏元素 |
| 🏆 | 获胜 / 最佳 | 结算 MVP |
| 🏳 | 投降 / 放弃 | giveup |
| 💰 | 金币 / 奖励 | economy |
| 📊 | 统计 / 数据 | status / profile |
| 📜 | 帮助 / 说明 | help |
| 👥 | 玩家 / 人数 | 玩家列表 |
| 🔒 | 私聊 / 权限 | 私聊提示、管理员 |
| 🐢 | 海龟汤专属 | 游戏标识 |

> 新增游戏时为游戏挑选**一个专属 emoji**（如 🎲、♟、🃏），登记在本表下方的"游戏专属"章节。

### 3.2 游戏专属 emoji

| 游戏 ID | Emoji | 说明 |
|---|---|---|
| `turtle_soup` | 🐢 | 海龟汤 |
| `trivia` | ❓ | 趣味问答 |
| `aoe3_battle` | ⚔️ | 帝国3斗蛐蛐 |
| `ra2_battle` | ⚔️ | 红警2斗蛐蛐（与帝国共用） |

### 3.3 工具专属 emoji

| 工具 | Emoji | 说明 |
|---|---|---|
| 今天吃什么 | 🍱 | 随机选餐 |
| 游戏王查卡 | 🃏 | 卡片查询 |
| 帝国3百科 | 🏰 | 兵种查询/对比 |
| 每日签到 | 📅 | 签到领金币 |

### 3.4 禁止使用

- ❌ 彩色文字 emoji 与语义 emoji 混用
- ❌ 表情叠加刷屏（如 `🎉🎉🎉🎉`）
- ❌ 使用生僻 emoji 导致某些客户端显示不出（如 `🫨` `🫷`）

## 4. 分隔符规范

```
━━━━━━━━━━━━━━━━━━━━━━━━━   主分隔（标题与正文之间、卡片顶部）
─────────────────────────   次分隔（卡片内段落之间）
···························   弱分隔（列表项之间、可选）
```

实际字符：
- 主分隔：`━` × 16（半角显示宽度相当于 32）
- 次分隔：`─` × 16
- 弱分隔：`·` × 18

`core.render` 提供常量：`render.SEP_HEAVY`、`render.SEP_LIGHT`、`render.SEP_DOT`。

## 5. 标题规范

**不使用** `═╔╗` 等双线框（部分客户端字体不等宽会破形），统一如下：

### 5.1 主标题（卡片顶部）

```
🐢 海龟汤 · 局号 A7F3
━━━━━━━━━━━━━━━━
```

### 5.2 副标题

```
📊 我的战绩
```

### 5.3 章节标题（卡片内）

```
▎玩法规则
```

（使用 `▎` 作为章节前缀，不再加粗——群消息无富文本）

## 6. 标准卡片模板

### 6.1 信息卡（`render.text_card`）

```
{emoji} {title}
━━━━━━━━━━━━━━━━
{body_line_1}
{body_line_2}
...
─────────────────
{footer_line}
```

**示例**：
```
🐢 海龟汤 · 局号 A7F3
━━━━━━━━━━━━━━━━
《午夜的第二杯咖啡》

深夜的咖啡馆里，陈默点了
两杯美式。他只喝了一杯，
然后对着第二杯哭了整整一
小时，最后笑着离开了。
─────────────────
💡 提问以 ? 结尾
💡 宣告汤底以「汤底:」开头
```

### 6.2 菜单卡（`render.menu`）

```
🎮 游戏大厅
━━━━━━━━━━━━━━━━
🐢 海龟汤
   1-10 人 · LLM 驱动
   /play turtle_soup
─────────────────
💰 金币：1250
📜 /help 查看帮助
```

### 6.3 列表卡（`render.list`）

每项一行，前缀编号或小 emoji：

```
📊 本群今日战绩
━━━━━━━━━━━━━━━━
🥇 张三        12 胜
🥈 李四         8 胜
🥉 王五         5 胜
  4. 赵六       3 胜
  5. 孙七       1 胜
```

### 6.4 结果卡（`render.result`）

```
🏆 游戏结束
━━━━━━━━━━━━━━━━
🐢 海龟汤 ·《午夜的第二杯咖啡》

结果：胜利 ✅
用时：18 分钟
提问：23 次
MVP：@张三

─────────────────
▎汤底
{truth_preview}
···
查看完整回顾请点击下方合并消息
```

后面紧跟一个**合并转发**包含完整问答回顾与汤底全文。

### 6.5 状态行（`render.status_line`）

不成卡片，用于 inline 反馈：

```
@张三 ❓ 陈默在等人吗？
↳ ❌ 不是
```

```
@李四 ❓ 陈默认识咖啡馆老板吗？
↳ 💡 关键线索：想想第二杯是为谁点的
```

## 7. @ 规则

- **群内需要回复特定玩家**：必须 `@昵称`，便于手机通知
- **群内广播**：不 @ 任何人
- **行动引导（"该你了"）**：`@昵称` + 换行 + 提示

`core.session.broadcast(..., at=qq_id)` 自动处理。

## 8. 错误与降级提示

| 场景 | 模板 |
|---|---|
| 输入非法 | `⚠️ 格式不对，示例：{example}` |
| 冷却中 | `⏱ 再等 {sec}s 就好` |
| 权限不足 | `🔒 这条命令需要 {role} 权限` |
| 余额不足 | `💰 金币不足（需要 {need}，当前 {balance}）` |
| LLM 失败 | `⚠️ 汤主走神了，请再问一次（内部错误已记录）` |
| 超时 | `⏱ 超时啦~ 本次跳过` |

## 9. 合并转发使用场景

QQ 的合并转发消息：群里显示为一条"折叠消息"，点进去才展开，**不刷屏且无长度限制**。

**使用场合**：
- 海龟汤结算后的完整问答回顾（几十条）
- `/help` 详细版
- `/profile` 历史战绩（10+ 条）

**不使用场合**：
- 单条信息量少时（直接发）
- 需要用户立刻看到的信息（合并转发要点击才展开）

`core.session` 提供 `send_forward(group_id, items)` 辅助构造。

## 10. `core.render` API 速览

```python
# 常量
render.SEP_HEAVY        # "━" * 16
render.SEP_LIGHT        # "─" * 16
render.SEP_DOT          # "·" * 18

# 基础原语
render.title(emoji: str, text: str) -> str
render.section(title: str) -> str                   # "▎xxx"
render.kv(label: str, value: str, width: int = 8) -> str
render.truncate(text: str, max_chars: int, suffix: str = "…") -> str

# 卡片
render.text_card(
    title: str,
    body: str | list[str],
    *,
    emoji: str = "",
    footer: str | list[str] | None = None,
) -> str

render.menu(
    title: str,
    items: list[MenuItem],
    *,
    footer: str | None = None,
) -> str

render.list_card(
    title: str,
    items: list[str],         # 已格式化的行
    *,
    emoji: str = "",
    footer: str | None = None,
) -> str

render.result(
    title: str,
    game_name: str,
    summary: dict,            # 会渲染成 kv 列表
    highlight: str | None = None,
    footer: str | None = None,
) -> str

render.status_line(
    actor: str,        # "@张三"
    action: str,       # "❓ 陈默在等人吗？"
    response: str,     # "❌ 不是"
) -> str

# 工具
render.paginate(items: list[str], page: int, per_page: int = 10) -> tuple[list[str], str]
# 返回（当前页项，页脚 "(1/3)"）
```

## 11. 示例：海龟汤一局的完整视觉流

**开局**：
```
🐢 海龟汤 · 局号 A7F3
━━━━━━━━━━━━━━━━
《午夜的第二杯咖啡》

深夜的咖啡馆里，陈默点了
两杯美式。他只喝了一杯，
然后对着第二杯哭了整整一
小时，最后笑着离开了。
─────────────────
💡 提问以 ? 结尾
💡 宣告汤底以「汤底:」开头
💡 /soup giveup 投降
💡 /soup status 查看进度
```

**过程**：
```
@张三 ❓ 陈默认识第二杯咖啡的对象吗？
↳ ✅ 是
```

```
@李四 ❓ 陈默是在等人吗？
↳ ❌ 不是
```

```
@张三 ❓ 第二杯是点给已逝去的人吗？
↳ 💡 关键线索：方向对了，但身份还差一点
```

**宣告**：
```
@张三 宣告：第二杯是陈默点给去世的父亲的，今天是父亲的忌日。
━━━━━━━━━━━━━━━━
🏆 汤底正确！

「汤底」陈默的父亲生前最爱这家咖啡馆的美式，
每年忌日陈默都会来点两杯：一杯自己喝，
一杯隔空敬父亲。今年他终于能笑着回忆起
父亲生前的温柔，所以笑着离开了。
```

**结算**（`render.result`）：
```
🏆 游戏结束
━━━━━━━━━━━━━━━━
🐢 海龟汤 ·《午夜的第二杯咖啡》

结果：  胜利 ✅
用时：  18 分钟
提问：  23 次
MVP：   @张三

奖励：  💰 +100 金币
─────────────────
完整问答回顾 👇
```

后接合并转发（问答全记录）。

## 12. 新增游戏接入本规范

在 `docs/games/<id>.md` 的"视觉清单"章节登记：
1. 游戏专属 emoji
2. 开局卡片样式
3. 交互反馈格式
4. 结算卡片字段
5. 是否使用合并转发、使用场景

**不得**在游戏代码里绕过 `core.render` 直接拼字符串输出正式消息。
（临时 debug print 除外）
